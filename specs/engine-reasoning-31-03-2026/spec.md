# Engine Reasoning & Multi-Agent Rework — 31-03-2026

## Problem Statement

The core engine treats LLMs as dumb action selectors and visual scanners, overriding their reasoning with rigid heuristics. This creates systemic blind spots:

- **Navigator** can see that an action had no effect (e.g., "button still shows Add to cart") but has no way to express that — its only options are retry or give up.
- **Review Agent** only receives a screenshot + action name + "ok/failed". It has no behavioral context (DOM state, intent, action trace), so it can only catch visual defects. Silent functional failures (button renders correctly but doesn't respond) are invisible to it.
- **Loop detection** is a hardcoded heuristic that forces an exit with `status: "passed"` — overriding the LLM's own reasoning and hiding real application bugs.
- **Status determination** uses a boolean formula (`meaningfulActions >= 3 && !hasFailedAssertions`) instead of the model's assessment of whether the intent was fulfilled.
- **Auth fallback** has an LLM backup, but it's a stripped-down mini-agent without a11y resolution, Stagehand, or proper error reporting.

**Evidence:** Run `6a630acb` — error_user on SauceDemo. Intent: "add three products and verify cart." Agent added 2 of 3 (3rd silently failed due to error_user restrictions). Agent noticed ("button still shows Add to cart") but couldn't report it. Review Agent saw a normal-looking page and found nothing. Loop detection forced exit with `"passed"`. Zero bugs reported. Summary acknowledged the issue but status was already locked to `"passed"`.

## Decisions

- **Multi-agent architecture stays.** Navigator navigates, Review Agent reviews. No `"bug"` action added to Navigator — that blurs responsibilities. Instead: Navigator gains an `observation` field to communicate behavioral signals, Review Agent gains the context to act on them.
- **Heuristics become advisory, not control flow.** Loop detection informs the LLM ("page state unchanged after your action"), LLM decides what to do. Hardcoded forced exits become a last-resort safety net at a higher threshold, returning `"failed"` — never `"passed"`.
- **Auth fallback uses the full agent.** `tryAgentAuth` replaced with a capped `runAgent` call. No separate mini-agent.
