# Batch Fix Spec — 27-03-2026

## Decisions

- **#2 (API auth) DROPPED** — local-first dockerized OSS tool; auth is unnecessary for default use. Common practice in similar tools (Playwright UI, Grafana OSS, n8n, Portainer) is no auth by default.
- **#1 (Run queue)** — Use BullMQ + Redis. Concurrency = auto-detect based on available memory (no hardcoded cap). Add Redis to docker-compose.
- **#4 (Credential encryption)** — AES-256-GCM with `ENCRYPTION_KEY` env var. When unset, falls back to unencrypted (backwards compatible).
- **#20 (Video/screenshot cleanup)** — Tied to run deletion: when a run is deleted, its video and screenshot files are deleted too.
- **#23 (Visual baseline comparison) DROPPED** — not needed at this stage; LLM-based review is the primary detection mechanism.
- **#27 (Path Generator)** — Remove step limit entirely. Let the Path Generator create full-fledged plans for total coverage, not artificially truncated to 5 or 8 steps.
- **#33 (Test suite)** — Focus on highest-risk areas only: regression engine, bug enrichment, a11y tree extraction, memory system.
